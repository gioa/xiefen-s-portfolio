Sphinx: A Browser-based Game Engine

Project Background:
The lack of smooth experience in a mobile HTML5 game is always a big problem for game developers. To make the performance of web game like that of a native game and save game developers from worrying about performance issue, we build Sphinx, a game engine particularly optimized for HTML5 games.


Role: Product Manager

Teams: Product, Operation, Engineering, Design, Marketing

Duration: 4 months on and off


Overview:

I was the product owner of Sphinx and I managed to deliver a web platform, an Android emulator and a packaging platform for our users.


The design process always starts from a problem statement. Take the Android emulator as an example, the developers do not know to which extent Sphinx can assist them. So the aim is to provide developers a tool for easily testing game performance on Sphinx engine.
Then we detailed user needs and defined the scope.
Next the alternatives of functional organization have been explored and 3 prototypes was drafted to test which structure served the user best.
After testing we selected a plan and moved to the detailed interface and graphic design to finalize the UI as below.
