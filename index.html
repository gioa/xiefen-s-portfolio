Design Process


Users and Information Architecture

Before jumping into the design work, I started to think about the users of the MHCID website and how to organize the current content to serve them best. I remembered when I applied for this program, I visited the website often. The website's information is clean and well organized into different sections, but the layout looks outdated.


Wireframe

To give the MHCID homepage a facelift, I first followed the current design trends, which includes large image, responsive design, and longer scrolling site with clear sections. So my 1st design starts with the logo and navigation bar on the top, followed by a large image (could be the background image for the logo and menu); then the notification section, the program description section with images to serve the content, and the news section; lastly, the footer has the general elements like logo, contact information, subscribe functions, and sponsor information. It also includes the navigation bar in case user wants to jump to other page after scrolling down.



I want to try different pattern for the second design. I really like google's material design. So I tried the flat design here. I hide the navigation in the left top corner, then I have a major area with different color to highlight the pages I want the user to click into. These are also the elements from the menu bar, with a bigger appearance to motivate users to click into. My inspiration comes from Google Design .


Visual Design


Next I move to css styling. I decided to do the first design since the elements did not change much. I also layout the visual design with photoshop first and then wrote the code.

Design of TV Applications

Opera TV Store


The Opera TV Store is an HTML5-based storefront displaying web applications made for TV and run from the cloud. It gives the user a comfortable, “lean-back” experience, using a standard TV remote control to navigate, select and launch apps easily. Like other stores, it includes various TV apps spanning from video (the most popular category), music, social networking, games, news and information.

My Contribution


My contributions are mainly focused on leading the outsourcing project of developing 10+ Chinese TV apps for Opera TV Store.

As the QA engineer in the TV team, I was responsible for moderating and publishing TV apps submitted by external developers or content partners. During this period I acquired TV apps design principles and understood potential developing issues. So when Opera TV Store planed to land on China TV market and required various local TV apps, I took charge and guided the development of various TV apps.

I also gave an speech on design of TV apps at Html5 Top conference 2012, to motivate HTML5 developers to build TV apps for Opera TV Store. The slides above is my presentation slides about design principles for TV screen.

Sphinx: A Browser-based Game Engine

Project Background


The lack of smooth experience in a mobile HTML5 game is always a big problem for game developers. To make the performance of web game like that of a native game and save game developers from worrying about performance issue, we build Sphinx, a game engine particularly optimized for HTML5 games.


My Contribution


I was the product owner of Sphinx and I managed to deliver a web platform, an Android emulator and a packaging platform for our users.


Checkout the website

Design Process


With clear objectives of building tools to promote the use of sphinx engine, I started the project by conducting competitive analysis on developer tools and defined the product scope, then generated and explored the alternatives of functional organization, with which I communicated often with stakeholders and internal users for feedback.

After that, I built different prototypes for user testing. After iterative testing, we selected a plan and moved to the detailed interface and graphic design to finalize the UI.


Reflection



Sphinx is more like an engineering directed project, with which I did not carry out the whole design process. I should start with users' needs research, and identify the project feasibility and objective before jumping into the design part.

Master Thesis: Context Aware Framework

Project Background


Context aware means applications can both sense and react based on the environment. But in reality the commercial context aware mobile applications are still limited, mostly are those using location information to find nearby friends or services available in the market.

This happened mainly because of a lack of an effective mechanism to collect and share contextual information. Nowadays, user context data are scattered around due to the significant growing in smart phones, mobile apps, sensing technology and social networking, how to combine these data to leverage information about users are important and promptly needed, which motivated the topic of my thesis: Applying Personal Cloud in Existing Mobile Context Aware Framework.

My work


In my thesis project I reviewed the approaches in existing context aware platforms, and reformed the context aware framework by absorbing previous fundamental technical aspects into personal cloud. By using a user-centric context collector approach, I enhanced the overall applications intelligence and reduced the privacy and security problem. A prototype application has also been developed for the purpose of evaluating the Context widget and obtaining hands-on experience of building a context-aware application. The function of the mobile application is fetching the content from a web site for film on TV today. And based on historical selected liked film database, contexts on user film preferences are auto-generated and context aware adaption is allowed for updating the film list order based on its reasoning logic.
